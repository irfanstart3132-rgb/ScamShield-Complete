# 🔒 Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ScamShield, please report it responsibly.

**DO NOT** create a public GitHub issue.

Instead, email us at:
📧 **irfanstart3132@gmail.com**

### What to Include
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Time
- **24 hours**: Acknowledgment of report
- **72 hours**: Initial assessment
- **7 days**: Fix implementation
- **14 days**: Public disclosure (if agreed)

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | ✅ Fully supported |
| < 1.0   | ❌ Not supported   |

## Security Best Practices

ScamShield follows these security principles:
- All API keys stored server-side only
- JWT authentication with rotation
- Input sanitization on all endpoints
- Rate limiting per user plan
- CORS protection enabled
- Regular dependency audits
